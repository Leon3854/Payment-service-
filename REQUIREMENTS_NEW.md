Техническое задание: Распределенная платежная экосистема (Phase 2: Highload & AI-Driven)
Цель проекта: Трансформация базового платежного сервиса в отказоустойчивую, масштабируемую инфраструктуру с интеллектуальным анализом транзакций и защитой от фрода.

1. Архитектурные изменения (The Core)
   Polyglot-подход:
   Node.js (NestJS): Выступает в роли API Gateway, управляет GraphQL-схемами, WebSockets и авторизацией.
   Go (Golang): Выделенный микросервис-воркер (payment-processor) для высокоскоростной обработки очереди транзакций.
   Messaging (RabbitMQ):
   Реализация асинхронного взаимодействия. Гарантия доставки At-least-once через Publisher Confirms и Dead Letter Exchanges (DLX) для обработки сбойных платежей.
   Orchestration: Подготовка Helm-чартов или K8s-манифестов для деплоя в кластер с настройкой горизонтального автоскалирования (HPA).
2. Надежность и Согласованность (Consistency & Security)
   Distributed Locking (Redis SET NX): Внедрение механизма блокировок для предотвращения Double-Spending (двойных списаний) при конкурентных запросах.
   Idempotency API: Реализация Idempotency-Key на уровне Middleware для всех мутаций, связанных с движением средств.
   Anti-N+1 Strategy: Рефакторинг финансовых отчетов в Prisma (Eager Loading) для минимизации нагрузки на PostgreSQL при построении сложных выписок.
3. Интеллект системы (AI-Native Layers)
   AI Anti-Fraud Filter: Интеграция с LLM (DeepSeek/Claude) через асинхронный поток для оценки риска транзакций (анализ аномалий в реальном времени).
   AI Categorization: Автоматическое присвоение категорий расходов (MCC-коды) на основе текстовых дескрипторов платежа с использованием нейросетей.
   Event-Driven AI Pipeline: Транзакция -> RabbitMQ -> AI-Worker -> DB Update -> WebSocket Push.
4. Наблюдаемость и CI/CD
   Health Checks: Настройка Liveness и Readiness проб для корректной работы в K8s.
   Automated QA: Покрытие критических сценариев (списания, возвраты, ошибки связи) интеграционными тестами на Jest.
   GitHub Actions: Пайплайн для сборки Docker-образов под разные архитектуры (multi-arch).
