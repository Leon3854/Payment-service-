### ТЗ: Универсальный микросервисный Billing Engine

## 1. Цель проекта

Разработка отказоустойчивого движка для управления платежами и подписками с поддержкой рекуррентных списаний, системы Dunning (взыскание задолженности) и интеграцией через RabbitMQ.

### 1. Побочные задания
Подготовить к горизонтальному масштабированию и интеграции с другими микросервисами, плюс автоматизацией развертывания и тестирования(CI/CD). 
Так же к движку должен быть подключен GraphQL API Gateway который должен подготовить к горизонтальному расширению( для снежениея нагрузки между копиями сервисов), защите и кешированиею частых запросов, аутентификацию, сжатие, маршрутизацию и логирование всех запросов в одно место. 
Данный GraphQL API Gateway надо разработать как подготовку к развитию и последующему расширению и маштабированию прокетка.  

## 2. Функциональные требования
# Core Payments:
Интеграция с ЮKassa (вебхуки, создание платежа, подтверждение).
Поддержка разовых и рекуррентных (автосписания) платежей.
Механизм идемпотентности (защита от двойных списаний через Redis).
Subscription Management: - управление подписками
Управление жизненным циклом подписки (PENDING, ACTIVE, PAST_DUE, CANCELED).
Автоматическое формирование инвойсов за N дней до списания.
Dunning System (6 стадий):
Автоматический ретри-механизм при неудачном списании.
Каскадная отправка уведомлений (Email -> SMS) через RabbitMQ.
Использование Delayed Message Exchange для планирования попыток.
Refunds & Sagas:
Возврат средств по запросу.
Компенсирующие транзакции: если подписка не активировалась, инициировать возврат денег.

## 3. Технический стек (согласно твоей структуре)
Framework: NestJS (Node.js).
Database: PostgreSQL + Prisma ORM.
Messaging: RabbitMQ (Quorum очереди + Delayed Message Plugin).
Caching: Redis (для идемпотентности и временных блокировок).
Infrastructure: Docker, Docker-compose (для оркестрации базы, кролика и сервисов), API Gateway (для маршрутизации внешних запросов).
4. Архитектурные требования
Event-Driven: Сервис оплат общается с сервисом подписок исключительно через события RabbitMQ.
Resilience (Отказоустойчивость):
Автоматическое переподключение к RabbitMQ/DB.
Prefetch limit = 1 для предотвращения перегрузки Node.js.
Observability: Централизованное логирование (через Logger NestJS) всех этапов транзакции.
