### 1. Общее описание
Разработать микросервис платежей (payment-service) на NestJS с поддержкой разовых и рекуррентных платежей, подписок, даннинга и возвратов. Сервис должен быть готов к масштабированию и интеграции с другими микросервисами, а так же с автоматизацией развертывания и тестирования(CI/CD). Так же к сервису должен быть подключен GraphQL API Gateway который должен подготовить к горизонтальному расширению( для снежениея нагрузки между копиями сервисов), защите и кешированиею частых запросов, аутентификацию, агрегацию запросов, сжатие, маршрутизацию и логирование всех запросов в одно место. Данный GraphQL API Gateway надо разработать как подготовку к будущему развитию проекта и добавлению новых микросервисов  

### 2. Технический стек
Backend: NestJS, TypeScript

База данных: PostgreSQL с Prisma ORM

Очереди: RabbitMQ для гарантированной доставки сообщений

Кэш: Redis для блокировок и идемпотентности

Контейнеризация: Docker, Docker Compose

CI/CD: GitHub Actions

Тестирование: Jest, Supertest

Платежные провайдеры: Yookassa (основной), с возможностью расширения

### 5. API спецификация
5.1. Основные эндпоинты
text
POST   /payments                 - Создание платежа
GET    /payments/:id             - Получение платежа
POST   /payments/:id/capture     - Подтверждение платежа
POST   /payments/:id/refund      - Возврат средств
GET    /payments/user/:userId    - История платежей пользователя

POST   /subscriptions            - Создание подписки
GET    /subscriptions/:id        - Получение подписки
POST   /subscriptions/:id/cancel - Отмена подписки
GET    /subscriptions/user/:userId - Подписки пользователя

POST   /invoices                 - Создание счета
GET    /invoices/:id             - Получение счета
GET    /invoices/user/:userId    - Счета пользователя

POST   /webhooks/yookassa        - Webhook от Yookassa
